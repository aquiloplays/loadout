# Loadout — project guide

Cross-platform stream toolkit for Streamer.bot, built by Clay (`aquiloplays`): one SB import bundle + one .NET DLL delivering 30+ modules (info commands, alerts, welcomes, the unified Bolts wallet, minigames, a dungeon mini-RPG, hype trains, quests/achievements, tip integration) plus an off-stream Discord bot, 30+ OBS overlays, and the local "Aquilo Bus" WebSocket event stream that every Aquilo product shares. This repo is the source of truth — see `HANDOFF.md` for the workstation-handoff picture (what lives here vs. on other machines/repos), `README.md` for the product tour, `CONFIG.md` for the canonical settings reference, and `CONTRIBUTING.md` for code style and the add-a-module/overlay/trigger recipes.

## Stack & deploys

- **`src/Loadout.Core/`** — the DLL: C# 9 targeting **net48** (builds from the .NET 8 SDK; net48 reference assemblies bundled), WPF settings/onboarding UI (code-behind, no MVVM), reflective boot from 9 inline-C# trampoline actions in `streamerbot/actions/`. **There is no dotnet toolchain in Linux agent containers — do not attempt to build the DLL; CI does it** (`build.yml` compiles on windows-latest for every push and PR).
- **`discord-bot/`** — Cloudflare Worker (plain JS modules, ~200 files at repo depth 1, `worker.js` entry): Discord slash commands, wallet sync, overlay canvas, Boltbound card engine, vault, and more. Bindings: KV (`LOADOUT_BOLTS` — shared with aquilo-site and the auth broker, be conservative — plus `STATE`), D1 (`aquilo_bot_db`, migrations as `*-migration.sql` files), Durable Objects (`OVERLAY_DO`, `ACTIVITY_DO`, `TTS_DO`), R2.
- **Python side-projects**: `aquilo-gateway/` (Discord Gateway shim on Railway, forwards gateway events to the Worker), `companion-streamkey/` (Windows tray app, PyInstaller, released via its own tag-driven workflow), `companion-kindle/` (DEPRECATED — superseded by `extension-kindle/`, a Manifest V3 browser extension).
- **Workflows** (`.github/workflows/`):
  - `build.yml` — compiles DLL + SB import bundle on every push/PR to `main` (the PR gate; keep it green)
  - `release.yml` — tag `v*` → Release build → draft GitHub Release on `aquiloplays/loadout-downloads` (the public binaries repo behind `download.aquilo.gg/loadout`)
  - `post-release-notes.yml` — release published → Discord embed via the SF bot service
  - `deploy-worker.yml` — push to `main` touching `discord-bot/**` → applies pending D1 migrations, then `wrangler deploy`. **Merging worker changes deploys them.**
  - `companion-streamkey-release.yml` — tag `companion-streamkey-v*` → single-file exe release
- Release runbook: `RELEASING.md` + `tools/release.ps1 -Version X.Y.Z -PushTag` (keeps csproj version, boot action, and tag in sync; CI fails the release if they diverge).

## Commands

Runnable in a Linux agent container (verified):

- `cd discord-bot && npm install` — the only installable deps in the repo (fast; the SessionStart hook does this on web sessions)
- `node discord-bot/test/test-<name>.mjs` — self-contained unit tests with in-memory KV/D1 mocks; one file per feature, 46 files. There is no runner/aggregate script. **Known-failing (pre-existing test drift, 12 of 46 as of 2026-07):** card-art-remix, character-pet, hero-death, hubs-and-schedule, onboarding, onboarding-admin, pet-leveling, pvp-combat, pvp-flow, scratch-flow, stream-squad, vote-hub. Don't treat those as regressions you caused; do run the test file next to whatever worker module you touch.

Windows-only (documented, not runnable here): `handoff.ps1` (one-shot build+bundle+install), `tools/build-dll.ps1`, `tools/build-sb-import.ps1`, `tools/install-dev.ps1`, `tools/release.ps1`, plus the sprite/art pipelines (`tools/*.ps1`, `tools/*.mjs`, root `build-*.py`).

There is no root package.json, no lint, no typecheck. `build.yml` on the PR is the only automated gate.

## Layout

```
src/Loadout.Core/       C# DLL: Modules/, Settings/, UI/, Bus/, Bolts/, Games/, Apex/,
                        Patreon/, Discord/ (sync client), Identity/, Workers/, Updates/
streamerbot/            SB import bundles (loadout/hangman/powerdeck/printerbot) + actions/
                        (9 inline-C# trampolines; bundles are REGENERATED by build scripts —
                        edit actions/ + tools/build-sb-import.ps1, not the .sb.txt)
discord-bot/            Cloudflare Worker (flat JS modules + test/ + *-migration.sql +
                        its own docs: CARD-ENGINE-MECHANICS.md, MOD-PERMISSIONS.md, ...)
aquilo-gg/              overlays/ (30+ OBS overlay frontends), sprites/ (generated art),
                        loadout/ (landing page), worker/loadout-link-worker.js (additive
                        routes for the shared SF Patreon worker), link/
aquilo-gateway/         Python Discord Gateway shim (Railway)
companion-streamkey/    Windows tray app for TikTok RTMP keys (Python/PyInstaller)
companion-kindle/       DEPRECATED — see extension-kindle/
extension-kindle/       Kindle-highlights browser extension (owner-only Knowledge Vault feed)
integrations/           StreamFusion Aquilo Bus client drop-in
tools/                  build/release/install/diagnostic PS scripts + sprite pipelines
assets/                 brand icon; root *.png files are design contact sheets/mockups
```

Architecture in one line: SB imports a 14 KB bundle of trampoline actions → they `Assembly.LoadFrom` Loadout.dll → the DLL runs the WPF UI, starts the Aquilo Bus on `ws://127.0.0.1:7470`, and dispatches normalized events to 24+ modules; overlays and other products subscribe to the bus.

## Conventions & gotchas

- **C# style** (full list in `CONTRIBUTING.md`): 4-space K&R, `_camelCase` private fields, `.ConfigureAwait(false)` on every non-UI await, module event handlers must never let exceptions reach the dispatcher (log to `Util.ErrorLog` and bail). Module registration order matters — readers of tracker state come after producers.
- **Everything defaults OFF**: new modules ship with their `ModulesConfig` toggle false and an onboarding checkbox.
- **Brand palette is shared with StreamFusion/aquilo.gg** (`#0E0E10` bg / `#3A86FF` accent / Segoe UI / 8 px radius); it lives in `UI/Styles.xaml`, `aquilo-gg/loadout/style.css`, and every overlay's CSS. Change one → change all.
- **Never edit generated bundles** (`streamerbot/*-import.sb.txt` / `.bundle.json`) by hand; regenerate via `tools/build-*.ps1`.
- **Patreon**: Loadout reuses StreamFusion's campaign and OAuth Worker; no feature is hard-gated on tiers (tiers = identity, early access, small bolts multiplier). `aquilo-gg/worker/loadout-link-worker.js` is additive to the SF worker — don't touch SF's existing routes.
- **discord-bot deploys on merge to main** (`deploy-worker.yml`) — worker changes are live minutes after merge; D1 migrations apply automatically, so name new migration files consistently (`<feature>-migration.sql`) and keep them idempotent-safe.
- **Secrets** live in GitHub repo secrets (`LOADOUT_DOWNLOADS_TOKEN`, `SF_RELEASE_POST_SECRET`, Cloudflare tokens) and per-machine state (`%APPDATA%\Loadout\`, DPAPI Patreon tokens, bus secret). Nothing secret is committed; keep it that way. `.dev.vars` / `.wrangler/` are gitignored.
- **Commit hygiene**: imperative subjects ≤72 chars, no emoji, one logical change per commit (enforced socially, see `CONTRIBUTING.md`).
- Overlays are query-string configured (`bus`, `secret`, `debug`); `?debug=1` renders a static demo without the bus — use it to sanity-check overlay changes.
- Test files in `discord-bot/test/` use hand-rolled in-memory mocks that pattern-match the SQL each module emits — if you change a module's queries, update its mock in lockstep.

## Claude Code sessions

`.claude/hooks/session-start.sh` installs `discord-bot/` npm dependencies when a web/remote session starts (the repo root has no package.json), and `.claude/settings.json` pre-approves running the discord-bot unit tests. The DLL cannot be built in the container — lean on `build.yml` CI for that.
