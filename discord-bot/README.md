# Loadout Discord bot

Always-on Discord bot for **off-stream Bolts engagement**. Viewers can check
their balance, gift bolts, view leaderboards, claim daily bonuses, and play
mini-games (coinflip, dice) right from your Discord server — even when your
stream is offline.

## Architecture

```
            ┌──────────────────────────────────────────┐
            │  Discord (slash commands)                │
            └───────────────────┬──────────────────────┘
                                │ HTTPS interactions endpoint
                                ▼
           ┌────────────────────────────────────────────┐
           │  Cloudflare Worker (this folder)           │
           │  - Verifies Discord Ed25519 signatures     │
           │  - Resolves /command → handler             │
           │  - Persists wallet state in CF KV          │
           │  - Runs minigame RNG                       │
           └────────────────────────────────────────────┘
                                ▲
                                │ HTTPS sync (push on save, pull on boot)
                                ▼
              ┌─────────────────────────────────────┐
              │  Loadout DLL (your Streamer.bot)    │
              │  Loadout.Discord.DiscordSync        │
              └─────────────────────────────────────┘
```

The Worker is the source of truth for wallet state in Discord. Loadout pushes
new wallet state up when the streamer hits Save (or when on-stream earns
happen), and pulls down before the stream goes live so off-stream earns are
reconciled. The Worker remains responsive 24/7 regardless of whether the
streamer's PC is on.

## Setup (streamer)

You don't need to host anything yourself. We run the shared Worker at
`https://loadout-discord.aquilo.gg/`. To wire it to your channel:

1. **Create a Discord application + bot** at <https://discord.com/developers/applications>:
   - New Application → name "Your Channel — Bolts"
   - Bot tab → Reset Token, copy it
   - OAuth2 → URL Generator → scopes `bot` and `applications.commands`
   - Permissions: `Send Messages`, `Use Slash Commands`
   - Copy the URL and invite the bot to your server.
2. **Register slash commands** (one-time, run from this folder):
   ```bash
   npm install
   APP_ID=... BOT_TOKEN=... node register-commands.js
   ```
3. **Set the interactions endpoint** in the Discord developer portal:
   - General Information → Interactions Endpoint URL = `https://loadout-discord.aquilo.gg/interactions`
4. **Open Loadout Settings → Discord bot tab**:
   - Paste your Discord application's public key (Developer Portal → General Information → Public Key)
   - Paste your server's Guild ID (right-click your server → Copy Server ID)
   - Click "Sync now" — this seeds the Worker with your current wallet state.
5. **Done.** Viewers can now run `/balance`, `/gift`, `/leaderboard`,
   `/daily`, `/coinflip`, `/dice`, and `/link` in your Discord.

## Self-hosting (optional)

If you'd rather run your own Worker (e.g. for privacy, or to avoid sharing
KV with other channels), this repo is fully deployable. You'll need a
Cloudflare account.

```bash
npm install
npx wrangler login
npx wrangler kv:namespace create LOADOUT_BOLTS
# Paste the returned id into wrangler.toml's [[kv_namespaces]] block.
npx wrangler deploy
```

Then point your Loadout settings at your worker URL and you're done.

## Slash commands

| Command | What it does |
|---|---|
| `/balance` | Show your bolts balance + lifetime earned + last earn |
| `/balance @user` | (mod-only) Show another viewer's balance |
| `/gift @user N` | Send N bolts to another viewer (cooldown applies) |
| `/leaderboard` | Top 10 wallet holders this server |
| `/daily` | Claim daily bonus (24h cooldown, streak multiplier) |
| `/coinflip N` | 50/50 — double your bet or lose it |
| `/dice N target` | Roll 1d6 — hit your target for 5x payout |
| `/link <platform> <username>` | Link your Discord identity to a stream platform username so off-stream earns reconcile when you're seen on stream |
| `/help` | Lists everything above |

## Wallet sync protocol

Loadout pushes/pulls wallet state via the Worker's `/sync` endpoint:

- `GET /sync/:guildId` — returns the current wallet snapshot. Loadout calls this
  on boot to merge Discord-side earns into the local wallet.
- `POST /sync/:guildId` — replaces the snapshot. Loadout calls this on save
  to push on-stream earns up.

Both are authenticated with a per-guild secret minted at registration time.
The secret is generated by Loadout and copied into the worker via
`/sync/:guildId/init` (one-time HMAC-protected call).
